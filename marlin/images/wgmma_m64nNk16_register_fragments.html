WGMMA .m64nNk16 / FP8 .m64nNk32 register fragment layout 速查

面向 PTX wgmma.mma_async 的两个常用浮点族： .f16/.bf16 使用 .m64nNk16， FP8 .e4m3/.e5m2 使用 .m64nNk32。 一个 warpgroup 的 128 个线程共同计算 D = A * B + D。

摘要基于 NVIDIA PTX ISA 9.2 文档，访问日期：2026-05-11。

1. 指令形状与类型

有效 N
8..256

步长为 8，即 N = 8 * i, i = 1..32。

乘法输入
f16 / bf16 / fp8

16-bit multiplicand 对应 k16；FP8 对应 k32。

register fragment 覆盖
A + D

B 必须通过 shared memory descriptor b-desc 提供。

2. 操作数整体形状

operand k16 形状 FP8 k32 形状 来源 register fragment 结论
A 64 x 16 64 x 32 a register tuple 或 a-desc shared memory descriptor 只有使用 a 时才用本页的 A register fragment layout。
B 16 x N 32 x N b-desc shared memory descriptor 没有 B register fragment；B 不作为 register tuple 传入。
D 64 x N 64 x N d register tuple 每个线程持有 N/2 个标量 D 元素。
下面所有 fragment 数量都是“每线程”的数量；整个 warpgroup 有 128 个线程。

3. A register fragment: 固定 64x16

A type 每线程寄存器 每线程标量元素 元素顺序
.f16 / .bf16 4 x f16x2 / 4 x bf16x2 8 a0, a1, ..., a7

总量校验：128 threads * 8 scalars = 1024 = 64 * 16。

每个 tid 的坐标公式

t = tid % 128 warp = floor(t / 32) lane = t % 32 r0 = 16 * warp + floor(lane / 4) r1 = r0 + 8 c = 2 * (lane % 4) a0,a1 -> A[r0, c:c+2) a2,a3 -> A[r1, c:c+2) a4,a5 -> A[r0, c+8:c+10) a6,a7 -> A[r1, c+8:c+10)

4. D register fragment: 64xN，按 8 列重复

D type 每线程寄存器 每线程标量元素 元素顺序尾部
.f16 N/4 x f16x2 N/2 d0, d1, ..., dX, dY, dZ, dW
.f32 N/2 x f32 N/2 X=N/2-4, Y=N/2-3, Z=N/2-2, W=N/2-1

D 的重复映射公式

t = tid % 128 warp = floor(t / 32) lane = t % 32 r0 = 16 * warp + floor(lane / 4) r1 = r0 + 8 p = lane % 4 for j in 0 .. N/8 - 1: c = 8 * j + 2 * p d[4*j + 0], d[4*j + 1] -> D[r0, c:c+2) d[4*j + 2], d[4*j + 3] -> D[r1, c:c+2)

5. FP8 指令：.m64nNk32

FP8 WGMMA 的 K 维度是 32，因此操作数整体形状是 A: 64x32， B: 32xN， D: 64xN。 它的 B 仍然只能来自 b-desc，A 可以来自 a-desc 或 register fragment a。

PTX 语法

wgmma.mma_async.sync.aligned.shape.dtype.atype.btype d, a-desc, b-desc, scale-d, imm-scale-a, imm-scale-b; wgmma.mma_async.sync.aligned.shape.dtype.atype.btype d, a, b-desc, scale-d, imm-scale-a, imm-scale-b; shape = .m64nNk32, N = 8*i, i = 1..32 atype = {.e4m3, .e5m2} btype = {.e4m3, .e5m2} dtype = {.f16, .f32}

A register fragment: 固定 64x32

A type 每线程寄存器 每线程标量元素 元素顺序
.e4m3 / .e5m2 4 x .b32 16 a0, a1, ..., a15

总量校验：128 threads * 16 scalars = 2048 = 64 * 32。

A 的坐标公式

t = tid % 128 warp = floor(t / 32) lane = t % 32 r0 = 16 * warp + floor(lane / 4) r1 = r0 + 8 c0 = 4 * (lane % 4) c1 = c0 + 16 a0..a3 -> A[r0, c0:c0+4) a4..a7 -> A[r1, c0:c0+4) a8..a11 -> A[r0, c1:c1+4) a12..a15 -> A[r1, c1:c1+4)

FP8 D register fragment

D type 每线程寄存器 每线程标量元素 布局
.f16 N/4 x f16x2 N/2 与第 4 节 D 布局相同，每 8 列重复一次。
.f32 N/2 x f32 N/2 与第 4 节 D 布局相同，每 8 列重复一次。
FP8 的 atype 和 btype 可以分别选择 .e4m3 或 .e5m2；这点不同于多数其它 floating point WGMMA 变体。

6. 读图时的几个关键点

结论 含义
以 warpgroup 为单位 4 个连续 warp 组成一个 warpgroup，tid % 128 决定 fragment 坐标。
A 随 K 变化，不随 N 变化 k16 的 A 是每线程 8 个 16-bit 元素；FP8 k32 的 A 是每线程 16 个 8-bit 元素。
D 随 N 线性增长 N 每增加 8，每线程 D 增加 4 个标量元素，也就是 2 个 f16x2 或 4 个 f32 寄存器。
B 不在 register fragment 表里 wgmma.mma_async 的 B operand 是 b-desc，对应 shared memory layout 和 matrix descriptor。
