LongCat-2.0 1.6T MoE 总参数 1.6T，每 token 激活约 48B，训练和大规模部署运行在 AI ASIC superpods 上。 meituan-longcat/LongCat-2.0， HF model card 仅 README / license / 图片；model card 标注权重 coming soon。 高

vLLM LongCat 模型适配 不是 2.0 专用，但覆盖 LongCat-Flash 模型与 MTP 适配。 vllm#23991； longcat_flash.py； longcat_flash_mtp.py LongCat-Flash 模型支持已 merge；未发现 2.0 专用 PR。 高
Transformers LongCat-Flash 提供 LongCat-Flash 的 HF Transformers 侧模型实现，可作为架构继承链参考。 transformers#40730； modeling_longcat_flash.py 已 merge；不是 LongCat-2.0 专用实现。 高
PD 分离、KVCache 传输、decode radix tree cache 面向 1M context 和 1.6T MoE serving 的系统级优化，包含 prefill/decode 分离、KV cache 并行与跨节点传输优化。 meituan-longcat/SGLang-FluentLLM； disaggregation； Quick_Start.md 官方开源 serving 分支；README 明确列出 layer-wise KVCache transfer 和 decode radix tree cache。 高
Speculative decoding: Eagle / MTP / PLD 博客提到 3-step MTP 和 CLI 复用索引，serving 侧需要低开销 speculative decoding。 SGLang-FluentLLM README； Quick_Start.md speculative 配置 官方 serving 分支支持 Eagle、MTP、PLD，并将 Target + Verify + Draft 合入单 CUDA graph。 高
MLA / FP8 KV cache / kernel 优化 长上下文 prefill/decode 的核心 kernel 栈，服务于 Attention I/O、计算和显存压力优化。 FlashMLA； DeepGEMM； flashinfer； SGLang-FluentLLM submodules 官方 kernel 仓库公开；SGLang-FluentLLM 以 submodule 固定 FlashMLA、DeepGEMM、DeepEP、EPS、FlashInfer。 高
EP / EPLB / zero expert / MoE routing decode 侧使用大 EP 度降低权重和专家 I/O，并通过异步 EPLB 解决专家负载不均。 SGLang-FluentLLM MoE layers； sglang#29515； DeepEP； EPS MoE/EP 相关代码公开；EPLB 与 2.0 线上放置策略未见完整公开实现。 中
6D 并行、超节点、ZeRO-1、Muon、1M context CP 训练系统优化：在 TP/CP/EP/DP/PP 外加入 EMBP，结合超节点、显存优化、长上下文 CP 和计算通信重叠。 未找到公开训练系统代码或 PR。 大概率是内部训练栈。 低
MOPD 多教师后训练 Agent Experts、Reasoning Experts、Interaction Experts 三类专家组，再通过 MOPD 融合能力。 未找到公开 post-training / MOPD 代码或 PR。 目前仅有技术说明。 低

检索结论

精确检索 LongCat-2.0 / LongCat 2.0 在 SGLang、vLLM、Transformers 的公开 PR，未发现模型实现 PR。公开 PR 多为 provider、pricing、model-list 或工具调用解析集成，不是 2.0 模型 serving 实现。

因此本页把 LongCat-2.0 的技术点分成三类：一是官方明确公开的 2.0 规格和技术描述；二是可代码追溯的 LongCat-Flash / Flash-Lite / SGLang-FluentLLM 继承链；三是当前未公开的训练和后训练内部栈。
