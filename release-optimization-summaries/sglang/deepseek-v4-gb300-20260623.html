DeepSeek-V4 on GB300 with SGLang 优化总结

来源：PyTorch Blog《Serving DeepSeek-V4 on GB300 with SGLang: 5x Higher Throughput at the Same Interactivity Since Day-0》，发布于 2026-06-23。整理日期：2026-06-24。

主线结论 同等交互性下，GB300 disaggregated 8K/1K 场景从 Day-0 no-MTP 约 2.2k tok/s/GPU 提升到 June 2026 MTP 约 11.2k tok/s/GPU。
核心方向 mHC/KV/MoE kernel、CUDA graph、PD+PP、DeepEP Waterfill、MTP 稳定性和 Dynamo DP 负载均衡。
主要上游 SGLang、Dynamo、DeepGEMM、NVIDIA GB300/Blackwell serving path。
证据边界 本文整理原文和 GitHub PR 元数据；没有重新跑 GB300 benchmark。
Kernel Fusion KV / SWA Memory CUDA Graph MTP Reliability
这篇文章的优化并不是单一 PR 带来的 5x，而是一组服务路径优化叠加：先让 DeepSeek-V4 在 SGLang 的 MTP/speculative 路径稳定运行，再压 mHC、KV compression、MoE 和 disaggregated decode 的关键开销，最后通过 CUDA graph 和 runtime recipe 把在线 serving 的 host gap 与 batch/concurrency 限制降下来。

优化点到 PR 的映射

类别 优化技术点 对应 PR / Commit 合入时间 作用
mHC kernel MHC pipeline 优化：mhc_pre 走 DeepGEMM，融合 norm，并加入 fused hc_head。 sglang#24775 2026-05-10 减少 mHC prefill/decode 中 kernel launch 和中间张量读写，是 DeepSeek-V4 路径的底层 kernel 提速。
mHC fusion 新增 mhc_fused_post_pre，把 mhc_post、pre-norm GEMM 和 mhc_pre finalization 的边界融合。 sglang#25976 2026-05-30 面向 decode 小 batch 降低 launch 开销；PR 自报总吞吐约 +3.35%。
KV compression 移植 KV Compression V2：包含 c4、c128、online c128，以及 fused norm/rope V2。 sglang#24890 2026-05-13 避免 DeepSeek-V4 高并发下压缩/indexer 路径成为瓶颈，支撑长上下文和高 batch 的 KV 压缩开销控制。
FP4 MoE DeepSeek-V4 W4A4 MegaMoE：把 activation 也从 MXFP8 切到 MXFP4。 sglang#25052 2026-05-14 降低 MoE activation footprint 和带宽压力，配合 DeepGEMM FP4 activation 支持提高高吞吐区间效率。
SWA memory disaggregated decode 中 full KV 和 SWA tail 分开预分配，不再按完整 prompt 过度预留 SWA KV。 sglang#24036, sglang#24857 2026-05-02 / 2026-05-13 让 decode worker 在同样显存下支撑更大 batch/concurrency，尤其适合 DeepSeek-V4 的 full attention + sliding window 混合 KV 形态。
CUDA graph DeepSeek-V4 DP attention 支持 breakable / piecewise CUDA graph。 sglang#25195 2026-06-08 把 mixed/prefill path 从 eager kernel launches 拉回 graph replay，减少 CPU host-side gap。
Speculative graph Eagle/speculative path 启用 breakable CUDA graph。 sglang@c4a7d12 2026-05-21 让 MTP/speculative draft worker 也初始化 piecewise CUDA graphs，补齐 speculative decode 路径的 graph 化。
PD + PP DeepSeek-V4 支持 pipeline parallelism，并接入 prefill/decode disaggregation。 sglang#24704 2026-05-16 让 mHC hidden state、SWA memory pool、PD KV pointer slicing 在 PP stage 下正确工作，是 GB300 多节点服务拓扑的基础能力。
DeepEP DeepSeek-V4 支持 DeepEP Waterfill，共享专家可作为额外 Waterfill expert，并支持 HashTopK/TopK balancing。 sglang#25391 2026-05-26 改善专家负载均衡和共享专家路径；PR sanity test 报告约 +2.6% 到 +4.31% 的吞吐收益。
PD-MTP metadata PD MetadataBuffer 使用 spec_hidden_size。 sglang#23919 2026-04-28 修复 prefill/decode 间 speculative hidden state 尺寸不匹配，稳定 MTP disaggregated serving。
SWA reliability disaggregated decode + MTP 下清理 stale SWA mappings，修复 SWA double-free。 sglang#25805 2026-05-22 避免 recycled full KV pages 携带旧 SWA 映射导致 double-free；PR 验证从约 2000 requests crash 改为 10240 requests 0 errors。
Warmup 按代表性 token-count bucket 预热 MHC kernels。 sglang#25810 2026-05-21 把稀有 bucket 首次触发的 20-40s 冷启动 stall 移到 startup warmup，降低 benchmark 和 live traffic 尾部抖动。
Blackwell fix fp8_einsum input scale 转 ue8m0，修复 Blackwell 上 V4-Pro NaN。 sglang#25733 2026-05-19 修复 B300/B200 上 autoregressive decode 的 NaN/garbage text；原文还指出这恢复了 MTP acceptance length。
Dynamo routing bootstrap_room 基于 prefill DP rank 生成。 dynamo#9080 2026-05-07 修复 Dynamo 不传 prefill_dp_rank 导致同一 prefill engine 内 DP rank 负载不均的问题。

没有逐项绑定 PR 的 runtime recipe

PD ratio

原文强调 prefill/decode ratio 调参。它影响每个 serving topology 的 batch、TTFT、TPOT 和 GPU 利用率，但文章没有给每个 ratio 绑定独立 PR。

Wide-EP

专家并行拓扑属于部署 recipe，与 DeepEP Waterfill、MoE kernel 和 GB300 网络/显存预算一起决定吞吐上限。

KV budget

full KV、SWA tail、token/request limits 的配比决定 decode worker 可承载的并发度；这类调参和 SWA prealloc 修复互相配合。

Tokenizer / flags

原文指向 SemiAnalysis InferenceX 配置和脚本作为复现入口，适合作为部署 recipe，而不是单个代码 PR 的归因。

后续路线

文章把后续工作集中在 DeepEP v2、ragged indexer、小 kernel fusion、SM120/SM121、NVFP4、cache/routing-sensitive path 和 agentic benchmark。SGLang 侧可继续跟踪 sglang#23602。

阅读这组 PR 时建议按依赖关系看：先看 PD/MTP 稳定性修复，再看 SWA/KV 内存容量优化，最后看 mHC/MoE/CUDA graph 的吞吐优化。否则容易把 5x 误解成某个单独 kernel PR 的结果。
