Kimi Blackwell Serving 优化调研

整理 SGLang、vLLM、TensorRT-LLM 在 Blackwell 系列 GPU 上服务 Kimi K2.5/K2.6/K2.7 的公开优化，覆盖 MLA backend、NVFP4/FP8 KV、MoE runner、EAGLE/DFlash、工具调用 parser 和部署修复。

Kimi K2.5 / K2.6 / K2.7 B200 / B300 / GB200 / GB300 MLA / NVFP4 / EAGLE / DFlash
主结论 Kimi K2 系列走 MLA/MoE serving 路径。
主线优化 MLA decode/prefill、FP8 KV、NVFP4 MoE、EAGLE/DFlash。
状态 部分条目是吞吐数据，部分是部署/正确性修复。
Kimi K2.5/K2.6/K2.7 的优化重点是 MLA decode/prefill、NVFP4/FP8 KV、MoE runner、speculative decoding、DFlash grammar decode。不同框架的公开资料需要区分吞吐提升、acceptance length、部署可用性和正确性修复，不能把这些效果混成同一种指标。

2. Kimi K2 Blackwell Serving 优化点

这里的 Kimi K2 指 K2.5/K2.6/K2.7-Code 这一支 MoE + MLA 架构。表格按框架记录公开优化手段、效果指标和来源链接。

框架 适用模型 / GPU 优化手段 公开效果 判断 来源
SGLang Kimi-K2.6 NVFP4；B200 / B300 / GB300 NVFP4 cookbook、Blackwell 推荐配置、EAGLE3/3.1 MLA draft、FP8 KV、tokenspeed MLA、MoE runner 调整。 8xB200 EAGLE3.1 MLA 验证中，GSM8K acc 0.980，平均 speculative accept length 3.20，质量与 HF reference 接近。 可落地 sglang#27714、sglang#26506、sglang#24441
SGLang Kimi-K2.7-Code；B300 / GB300 K2.7 cookbook、K2.6-compatible deployment、DFLASH grammar constrained decoding、JSON schema/regex decode、FP8 KV 配置修正。 DFLASH PR 在 Kimi-K2.7-Code TP4 B300 上验证 JSON schema/regex 有效，主要是功能/可用性验证，不是完整吞吐 A/B。 验证中 sglang#28064、sglang#28201、sglang#28943
vLLM Kimi-K2.6 NVFP4；GB200 / B200 / RTX PRO 6000 Blackwell EAGLE3/3.1、MLA + DCP、TRITON_MLA、FP8 KV、CUDA graph、NCCL graph XML、Blackwell fp8 tuning。 vLLM EAGLE3.1 blog 给出 SPEED-Bench per-user output throughput：并发 1 为 2.03x，并发 4 为 1.71x，并发 16 为 1.66x。 强相关 vLLM EAGLE3.1 blog、vllm#40608、vllm#40750
vLLM Kimi-K2.6；8xHGX B200 EAGLE3 draft、FP8 KV、关闭 prefix caching、fuse_allreduce_rms、MoonViT data parallel。 Lambda 公开数据：8192 in / 2048 out、32 concurrency、512 prompts 下 generation throughput 1408 tok/s，total throughput 7046 tok/s。 参考价值高 Lambda Kimi K2.6 model benchmark
vLLM Kimi-K2.7 INT4 / NVFP4；SM120 为 INT4 + FP8 KV / NVFP4 自动路径修复 Triton shared memory OOR，通过保守 num_stages=1 fallback 启动。 报告者确认 Kimi K2.7 可正常 load/deploy，但该 PR 不是吞吐优化 A/B。 可用性修复 vllm#46721、vllm#46728
TensorRT-LLM Kimi-K2.6 NVFP4 + Eagle3；B200 TensorRT-LLM runtime、Eagle speculative config、draft length 3、TP4 serve。 NVIDIA HF card 给出 draft length 3 时 MT-Bench average acceptance length 2.62，SPEED-Bench qualitative 2.67。 可复现入口清晰 nvidia/Kimi-K2.6-Eagle3
TensorRT-LLM Kimi-K2.5 / K2.6；GB200 / B300 修复 EAGLE3 draft arch registry、Kimi rejection sampling、post-norm / per-aux fc_norm 架构支持。 GB200 NVL72 4 GPU 上 Kimi-K2.5 NVFP4 + Eagle3 可以 end-to-end 生成；B300 rejection sampling crash 有修复。 部署修复为主 TensorRT-LLM#15426、TensorRT-LLM#15182、TensorRT-LLM#14988
CoreWeave / NVIDIA Kimi-K2.7-Code；GB300 / GB200 Custom NVFP4 quantization、DFlash speculator、D-PACE calibration、production endpoint 默认 NVFP4 + DFlash。 CoreWeave 博客称 K2.7-Code 在减少约 30% reasoning tokens 的同时针对 GB300/GB200 做了 serving 优化；D-PACE 带来约 8% acceptance length 改善。 生产信号 CoreWeave Kimi K2.7-Code blog、NVIDIA DFlash blog
